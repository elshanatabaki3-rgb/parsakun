    ۱) تست سریع داخل همین مرورگر (BroadcastChannel) — همین الان کار می‌کند

    این فایل از BroadcastChannel API استفاده می‌کند که به دو تب باز از یک مرورگر اجازه می‌دهد آنی با هم پیام مبادله کنند؛ بدون نیاز به اینترنت یا سرور. برای تست:

    • این فایل HTML را یک‌بار باز کن و یک اتاق بساز.
    • یک تب جدید باز کن (ترجیحاً با «تکرار تب» یا Ctrl+Click روی همین فایل) و کد اتاق را وارد کن.
    • توجه: اگر فایل را با دابل‌کلیک (آدرس file://) باز کرده‌ای، بعضی مرورگرها (به‌خصوص Firefox) هر تب را در یک «origin» جدا در نظر می‌گیرند و BroadcastChannel/LocalStorage بین تب‌ها مشترک نمی‌شود. برای رفع این مشکل، فایل را با یک سرور لوکال سرو کن، مثلاً:
    cd پوشه‌ای‌که‌فایل‌در‌آن‌است
python -m http.server 8000
# سپس در مرورگر باز کن:
http://localhost:8000/tabbeh-baz.html

    این روش فقط بین تب‌ها/پنجره‌های همان مرورگر روی همان دستگاه کار می‌کند، نه بین دو دستگاه مختلف.

    ۲) بازی واقعی بین دو دستگاه/شبکه‌ی متفاوت (WebRTC)

    برای ارتباط واقعی بین دو دستگاه در دو شبکه‌ی مختلف، باید از WebRTC استفاده کنی. مرورگرها برای برقراری اتصال مستقیم (Peer-to-Peer) نیاز دارند اطلاعات اتصال (Offer / Answer / ICE Candidates) را یک‌بار از طریق یک کانال جداگانه به نام «سیگنالینگ» مبادله کنند؛ بعد از آن، داده‌های بازی مستقیماً بین دو مرورگر جابه‌جا می‌شود (بدون عبور از سرور). دو روش رایج برای پیاده‌سازی سیگنالینگ:

    روش الف — کپی‌پیست دستی (بدون هیچ سروری):

    • بازیکن اول یک RTCPeerConnection می‌سازد، یک Offer تولید می‌کند و صبر می‌کند ICE کامل شود.
    • متن Offer (یک رشته‌ی JSON/SDP) را کپی و از طریق تلگرام/واتساپ برای دوستش می‌فرستد.
    • بازیکن دوم آن را در فیلد مخصوص Paste می‌کند، یک Answer تولید می‌شود و آن را برای بازیکن اول می‌فرستد.
    • بازیکن اول Answer را Paste می‌کند؛ از این لحظه یک RTCDataChannel مستقیم بین دو مرورگر باز می‌شود و دقیقاً همان پیام‌هایی که الان با channel.postMessage(...) فرستاده می‌شوند را می‌توان با dataChannel.send(...) فرستاد.
    // نمونه‌ی بسیار ساده (نیاز به تکمیل و رابط کاربری برای کپی/پیست دارد)
const pc = new RTCPeerConnection({ iceServers:[{urls:'stun:stun.l.google.com:19302'}] });
const dc = pc.createDataChannel('game');
dc.onmessage = e => console.log('پیام دریافتی:', e.data);

const offer = await pc.createOffer();
await pc.setLocalDescription(offer);
// صبر کن تا pc.iceGatheringState === 'complete' شود، سپس:
const offerText = JSON.stringify(pc.localDescription);
// این متن را برای دوستت بفرست (کپی/پیست)

    روش ب — سرور سیگنالینگ کوچک + Ngrok:

    • یک سرور WebSocket کوچک با Node.js بساز که فقط پیام‌های دو طرف یک «اتاق» را به هم پاس می‌دهد.
    • با دستور ngrok http 3000 آن را به یک آدرس عمومی (https) تبدیل کن.
    • هر دو بازیکن از طریق همان آدرس Ngrok به سرور وصل می‌شوند، Offer/Answer/ICE را از طریق آن مبادله می‌کنند و سپس اتصال مستقیم WebRTC برقرار می‌شود.
    // server.js — یک سیگنالینگ بسیار ساده
const { WebSocketServer } = require('ws');
const wss = new WebSocketServer({ port: 3000 });
const rooms = {};
wss.on('connection', ws => {
  ws.on('message', raw => {
    const msg = JSON.parse(raw);
    rooms[msg.room] = rooms[msg.room] || [];
    rooms[msg.room].push(ws);
    rooms[msg.room].forEach(client => {
      if (client !== ws) client.send(raw);
    });
  });
});

    روش ج — Firebase Realtime Database به‌جای سرور:

    • یک پروژه‌ی رایگان Firebase بساز و Realtime Database را فعال کن.
    • به‌جای WebSocket، Offer/Answer/ICE را در مسیر rooms/<کد اتاق> بنویس و با onValue(...) آن را در طرف مقابل بخوان.
    • بعد از برقراری اتصال WebRTC، دیگر نیازی به Firebase نیست؛ داده‌های بازی مستقیم جابه‌جا می‌شوند.

    نکته: ساختار کد این فایل به‌گونه‌ای نوشته شده که تمام ارتباطات از یک تابع مرکزی (saveAndBroadcast) عبور می‌کنند؛ اگر بخواهی WebRTC واقعی اضافه کنی، کافی است در آن تابع به‌جای channel.postMessage(...) از dataChannel.send(...) استفاده کنی.

    ۳) محدودیت‌های نسخه‌ی فعلی
    • نسخه‌ی فعلی برای بازی بین دو تب/پنجره‌ی همان مرورگر روی همان دستگاه به‌صورت کامل کار می‌کند.
    • برای بازی بین دو دستگاه واقعی (مثلاً گوشی تو و گوشی دوستت)، باید یکی از روش‌های WebRTC بالا را اضافه کنی.
    • هیچ داده‌ای به سروری ارسال نمی‌شود؛ تمام اطلاعات (وضعیت بازی، آمار، چت) فقط در مرورگر خودت در localStorage ذخیره می‌شود.
